# Power BI Dashboard Setup

The Power BI report connects directly to **Delta Lake parquet files** generated by the
PySpark notebooks. No database connection is required — Power BI reads the Parquet format natively.

---

## Loading the Delta Parquet Files

1. Open **Power BI Desktop**
2. Click **Home → Get Data → Parquet**
3. Browse to the unified Delta table folder:
   ```
   spark_docker\notebooks\delta_lake\
   ```
4. Select the `.parquet` file inside that folder and click **Open**
5. Repeat for individual tables if needed:
   - `spark_docker\notebooks\delta_hubspot\`
   - `spark_docker\notebooks\delta_mailchimp\`

---

## Dashboards in the Report

| Dashboard | Key Metrics |
|---|---|
| **Campaign Performance** | Open rate, click rate, unsubscribe rate by campaign |
| **Customer Lifetime Value (CLV)** | CLV by lifecycle stage and lead source |
| **ROI Analysis** | Revenue vs spend per campaign |
| **Sales Forecasting** | Deal value trends over time |
| **Contact Segmentation** | Contact distribution by company and lifecycle stage |
| **Revenue by Channel** | Breakdown of revenue across marketing channels |

---

## Refreshing Data

After re-running the full pipeline (ingest → clean → join → Delta Lake):

1. Open the `.pbix` file in Power BI Desktop
2. Click **Home → Refresh**
3. All visuals update from the latest parquet files

---

## Notes

- The `.pbix` file is excluded from Git due to binary size. Recreate dashboards by
  following the steps above after running the pipeline.
- Power BI **Pro** is only required if you want to publish to Power BI Service (cloud).
  All local analysis works with the free **Power BI Desktop**.
